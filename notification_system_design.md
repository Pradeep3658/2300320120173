# Notification System Design

## Overview
A simple notification system coordinating a logging middleware, a backend service, and a frontend app to deliver user notifications reliably.

## Components
- logging_middleware: centralized logging and metrics
- notification_app_be: API, notification dispatcher, persistence
- notification_app_fe: UI for subscribing and viewing notifications

## High-Level Flow
1. Events generated by app services are sent to the backend API.
2. Backend persists and enqueues notifications for delivery.
3. Logging middleware records events and delivery attempts.
4. Frontend subscribes via WebSocket or polls for new notifications.

## Technologies (suggested)
- Backend: Node.js/Express or Python/FastAPI
- Queue: Redis or RabbitMQ
- Frontend: React or Vue
- Logging: structured JSON logs, optional ELK stack

## Reliability
- Retry policy for failed deliveries
- Dead-letter queue for problematic messages
- Idempotent message processing
